Technical Limits Policy

To ensure the best possible experience for all Spica customers, we enforce technical limits. If you do not agree with the technical limitations described below then do not use any of the Managed Spica services. Teams are subject to the following cumulative limits:

Metric Max Measure
API Requests 6 per second per project
Buckets 25 per project
Fileds 50 per bucket
Functions 100 per project
Subscriptions 10 per project
Value 5000 characters per field
Asset size 100 MB per asset
Request size 1 MB
Response size 10 MB

If your organization requires higher limits, you can upgrade to an Enterprise tier, which allows for customizing individual limits. If your Organization's usage exceeds the technical limits, we reserve the right to take corrective action, including but not limited to the throttling of API or suspending/terminating your account.
